Multi-Agent Python Tutor

Mijn bachelorproef: een curriculum-bewust AI-tutorsysteem dat draait als Discord-bot. Multiple orchestrated agents deliver structured Python lessons, generate exercises, evaluate submitted code and adapt instruction to each learner's progress.

Het probleemThe problem

Generieke chatbots kunnen Python-vragen beantwoorden, maar ze geven geen les: geen curriculum, geen geheugen van wat je al beheerst, geen oefeningen op jouw niveau, geen structuur om je door de stof te loodsen. Ik wilde onderzoeken of een multi-agent architectuur binnen een Discord-bot een echte, zelfsturende leerervaring kan bieden — in plaats van een reactieve assistent te blijven. Het werd de onderzoeksvraag van mijn bachelorproef: “Hoe kan een multi-agent, agentic AI-systeem geïmplementeerd worden binnen een Discord-bot om zelfsturend programmeeronderwijs autonoom te begeleiden via natuurlijke taal?”Generic chatbots can answer Python questions, but they don't teach: no curriculum, no memory of what you've mastered, no exercises matched to your level, no structure to guide you through the material. I wanted to investigate whether a multi-agent architecture inside a Discord bot could deliver a real, self-directed learning experience — instead of remaining a reactive assistant. That became my thesis research question: “How can a multi-agent, agentic AI system be implemented within a Discord bot to autonomously guide self-directed programming education through natural-language interaction?”

De aanpakThe approach

Ik heb het systeem opgebouwd in vier lagen: een Discord-frontend, een FastAPI-backend, een LangGraph multi-agent orchestratielaag en een aparte code-executie-omgeving. De Discord-bot beheert per gebruiker een eigen les- en statuskanaal en stuurt berichten door naar de backend, die de sessiestate bijhoudt en de juiste agent aan het werk zet.I built the system in four layers: a Discord frontend, a FastAPI backend, a LangGraph multi-agent orchestration layer, and a separate code-execution environment. The Discord bot maintains per-user lesson and status channels and forwards messages to the backend, which tracks session state and dispatches the right agent.

De kern is een netwerk van zeven gespecialiseerde agents, gecoördineerd door LangGraph:The core is a network of seven specialised agents coordinated by LangGraph:

  • Intent classificeert wat de gebruiker wil (lesgeven, oefening, curriculumaanpassing, hulpvraag, ...)Intent classifies what the user wants (learn, exercise, curriculum change, question, …)
  • Router stuurt de state naar de juiste specialist op basis van de gedetecteerde intentieRouter dispatches to the right specialist based on the detected intent
  • Curriculum genereert en beheert dynamisch de topic-lijst — de gebruiker kan onderwerpen toevoegen, verwijderen, herordenen of skippenCurriculum generates and maintains the topic list dynamically — the user can add, remove, reorder or skip topics
  • Instruction levert de les stap voor stapInstruction delivers the lesson step by step
  • Help beantwoordt vervolgvragen tijdens de les zonder de leservaring te verstorenHelp answers follow-up questions during a lesson without breaking the lesson flow
  • Exercise genereert oefeningen die passen bij het huidige topic en niveauExercise generates exercises matched to the current topic and level
  • Evaluator voert de ingediende code uit in de sandbox, vergelijkt met verwachte output en genereert gerichte feedbackEvaluator runs submitted code in the sandbox, compares against expected output and generates targeted feedback

Alle agents draaien op llama-3.1-8b-instant via Groq, gekozen voor snelheid boven ruwe capaciteit — latency is doorslaggevend in een Discord-conversatie. Elke prompt volgt hetzelfde patroon (rol → context → regels → outputformaat), en de temperatuur verschilt bewust per taak: 0 voor JSON-output (curriculum, oefeninggeneratie, intent), 0.1–0.4 voor natuurlijke tekst zodat herhaalde interacties niet identiek voelen. JSON-prompts bevatten een expliciete schema-instructie plus een regex-extractie in Python als vangnet — die combinatie maakte de gestructureerde output betrouwbaar in de praktijk.All agents run on llama-3.1-8b-instant via Groq, chosen for speed over raw capability — latency matters more than benchmark quality in a Discord conversation. Every prompt follows the same pattern (role → context → rules → output format), and temperature varies deliberately by task: 0 for JSON-emitting agents (curriculum, exercise generation, intent), 0.1–0.4 for natural-language agents so repeated interactions don't feel identical. JSON prompts embed an explicit schema plus a regex fallback in Python — that combination made structured output reliable in practice.

Bewust géén gebruik gemaakt van LLM-native tool calling of MCP. Het model genereert alleen tekst; alle uitvoering (sandbox, curriculum-updates, sessie-state, progressie-opslag) gebeurt in de omringende Python-code. LangGraph bepaalt op basis van de sessie-state welk agent-knooppunt draait. De koppeling tussen prompttekst en code is soms strak — de evaluator moet bijvoorbeeld letterlijk de zin you are ready! uitspreken voor de Python-code een oefening als geslaagd markeert — maar die expliciete triggerzin voorkomt dubbelzinnige pass/fail-detectie.Deliberately no LLM-native tool calling or MCP. The model only generates text; all execution (sandbox, curriculum updates, session state, progress storage) lives in the surrounding Python code. LangGraph decides which agent node runs based on session state. The coupling between prompt wording and code logic is sometimes tight — the evaluator must literally emit the trigger phrase you are ready! before the Python code marks an exercise as passed — but that explicit contract avoids ambiguous pass/fail detection.

Het resultaatThe result

Een werkend prototype dat aantoont dat een multi-agent architectuur zelfsturend programmeeronderwijs kan ondersteunen: het genereert dynamisch een curriculum op basis van de leerdoelen, geeft les stap voor stap, laat de leerling het curriculum tijdens de sessie aanpassen, genereert oefeningen op maat van het huidige topic, voert code uit en geeft directe, gerichte feedback. De positionering zit doelbewust tussen statische e-learning en generieke chatbots: meer structuur dan een chatbot (curriculum, progressie, evaluatie), meer flexibiliteit dan een klassieke cursus.A working prototype that demonstrates a multi-agent architecture can support self-directed programming education: it dynamically generates a curriculum from the learner's goals, delivers lessons step by step, lets the learner reshape the curriculum mid-session, generates topic-matched exercises, executes code and returns immediate, targeted feedback. It deliberately sits between static e-learning and generic chatbots — more structure than a chatbot (curriculum, progress, evaluation), more flexibility than a classical course.

Interessante emergente eigenschappen: het systeem weigert off-topic verzoeken (routers fallback-pad), en jailbreak-pogingen worden als “niet-begrepen” behandeld — niet door expliciete guardrails, maar omdat de intent classifier de adversariële input simpelweg niet matcht op een bekende intentie. Dat is geen robuuste beveiliging (de thesis erkent dit expliciet), maar het is een aardig neveneffect van de architectuur.Interesting emergent behaviours: the system refuses off-topic requests via the router's fallback, and jailbreak attempts get treated as “not understood” — not through explicit guardrails, but because the intent classifier fails to match adversarial input to any known intent. Not robust security (the thesis is honest about this), but a nice side effect of the architecture.

Wat ik geleerd hebWhat I learned

Sterkere modellen leveren niet automatisch een beter product. In de praktijk hadden consistentie, snelheid en het ontwerp van de workflow meer impact dan pure benchmark-intelligentie. Een klein, snel model met een strakke prompt-structuur en duidelijke code-side contracten deed het beter dan “gewoon GPT-4 erop plakken” ooit had gedaan.Stronger models don't automatically make a better product. In practice, consistency, speed and workflow design mattered more than raw benchmark intelligence. A small, fast model with a tight prompt structure and clear code-side contracts outperformed anything “just glue GPT-4 on it” would have delivered.

Educatieve kwaliteit vraagt bewuste structuur. Zonder curriculum, prerequisite-logica en progressiebeheer verwordt zelfs een krachtig conversatiemodel tot een reactieve assistent. Structuur was het verschil tussen “een chatbot die Python-vragen beantwoordt” en “een tutor die je door de stof leidt.”Educational quality requires deliberate structure. Without curriculum, prerequisite logic and progress management, even a powerful conversational model collapses into a reactive assistant. Structure was the difference between “a chatbot that answers Python questions” and “a tutor that walks you through the material.”

Iteratief testen is niet optioneel. Veel ontwerpbeslissingen — de exacte triggerzin van de evaluator, de temperatuur per agent, de fallback van de router — werden pas duidelijk na echte interacties. Papieren ontwerpkeuzes overleven zelden het eerste contact met een echte gebruiker.Iterative testing isn't optional. Many design decisions — the evaluator's exact trigger phrase, per-agent temperature, the router's fallback behaviour — only became clear after real interactions. Paper design choices rarely survive first contact with a real user.

Eerlijk zijn over de grenzen. De sandbox is niet echt geïsoleerd — code draait in het host-proces met volledige builtins. Voor een onderzoeksprototype prima; voor productie ontoereikend. Ook de schaal-eigenschappen zijn niet gevalideerd voorbij één gebruiker, en het systeem antwoordt uitsluitend in het Engels ongeacht de input-taal. Deze beperkingen zijn expliciet gedocumenteerd in de thesis, met concrete aanbevelingen voor containerized execution, een persistente database, en een LLM-router voor provider-onafhankelijkheid.Being honest about the edges. The sandbox isn't truly isolated — code runs in the host process with full builtins. Fine for a research prototype; not fit for production. Scalability beyond a single user is unvalidated, and the system replies only in English regardless of input language. These limitations are explicitly documented in the thesis, alongside concrete recommendations for containerised execution, a persistent database, and an LLM router for provider independence.
